# Reference-Guided Wearable Heart-Sound Denoising

EE 596 final project — denoising phonocardiogram (PCG) recordings captured by a
wearable microphone, using a clean stethoscope baseline as a soft reference.

The core idea: motion and speech contaminate wearable PCG, but the *shape* of
the heart sound (S1/S2 transients on a quiet background) is well predicted by
a phase-aligned baseline recorded in a quiet posture. We exploit that with a
smooth composite objective and solve it with two solvers (gradient descent and
a custom majorization–minimization scheme), then compare them.

## Method

We estimate the clean PCG `x` from the noisy observation `y` by minimizing

```
F(x) = 0.5 * ||x - y||^2
     + lambda_g * sum_g sqrt( ||(D x)[g : g+K-1]||^2 + eps^2 )
     + lambda_r * sum_i  w_i * huber_delta(x_i - z_i)
```

- **Data fidelity** `0.5 ||x - y||^2` keeps `x` close to the observation.
- **Overlapping group sparsity (OGS)** on the first-difference `D x` promotes
  *piecewise-smooth* segments separated by sharp S1/S2 transients
  (Selesnick & Chen 2013). The `eps`-smoothing makes the penalty differentiable
  so we can use first-order solvers.
- **Robust reference attraction** pulls `x` toward a phase-aligned quiet
  baseline `z`, but the Huber loss caps the influence of large mismatches so
  motion artifacts in `y` cannot drag `x` toward an arbitrary reference.

### Solvers

Both solvers are implemented in [pcg_denoise.py](pcg_denoise.py):

- `solve_gd` — monotone gradient descent with Armijo backtracking line search.
- `solve_mm` — majorization–minimization. Each outer step builds a quadratic
  upper bound (linear majorizer for the concave OGS-in-energy term, and the
  half-quadratic majorizer for the Huber loss) and minimizes it via a
  symmetric tridiagonal solve in **O(n)** using `scipy.linalg.solveh_banded`.
  Monotone non-increasing by construction.

A finite-difference gradient check (`check_gradient`) is included so the
analytical gradient can be verified before tuning hyperparameters.

python3 -m venv .venv

## Running the demo

Default: use `mic_walking.wav` as the noisy observation and `mic_speaking.wav`
as the reference (a warm-up proxy — in production, the reference would be a
clean stethoscope recording).

```bash
python demo.py
```

Common options:

```bash
# Pick which files act as observation / reference
python demo.py --noisy data/mic_walking.wav --ref data/stethoscope_sit_quiet.wav

# Disable the reference term (lambda_r = 0); pure OGS denoising
python demo.py --no-ref

# Tune regularization
python demo.py --lambda-g 0.05 --lambda-r 0.5 --K 50
```

The demo writes:

- `results/denoise_demo.png` — overlay of `y` / `x_hat`, GD vs MM, convergence
  curves, and the spectrogram difference (`PSD_y − PSD_x`) showing what was
  removed.
- `results/denoised_mm.wav` — the MM-denoised waveform at 2 kHz.

## Sanity checks

Run `pcg_denoise.py` directly to verify the analytic gradient and confirm MM
monotonicity on a synthetic signal:

```bash
python pcg_denoise.py
```

## Hyperparameters at a glance

| Symbol     | Meaning                                 | Default              |
| ---------- | --------------------------------------- | -------------------- |
| `lambda_g` | OGS group-sparsity weight on `D x`      | `0.05`               |
| `lambda_r` | Reference-attraction weight             | `0.5`                |
| `K`        | OGS group size in samples (≈25 ms @ 2 kHz) | `50`              |
| `eps`      | OGS smoothing constant                  | `1e-3`               |
| `delta`    | Huber knee                              | `3 × MAD(y)`         |
| `FS_TARGET`| Working sample rate                     | `2000` Hz            |
| `PCG_BAND` | Pre-bandpass for the PCG band           | `25 – 150` Hz        |
